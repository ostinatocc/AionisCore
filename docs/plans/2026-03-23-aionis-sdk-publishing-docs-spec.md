# Aionis SDK Publishing Docs Spec

## Goal

Turn the SDK repository surface from internal package work into an external-facing release surface by adding:

1. a direct quickstart
2. a direct publishing checklist
3. clearer SDK README install and usage guidance

## Requirements

1. External developers should be able to find install and quickstart instructions without reading internal plan docs.
2. SDK release operators should have a single publishing checklist.
3. Root README should point directly to SDK quickstart and publishing docs.

## Non-Goals

1. Building a dedicated docs app.
2. Automating npm release.
3. Expanding the SDK v1 surface.
