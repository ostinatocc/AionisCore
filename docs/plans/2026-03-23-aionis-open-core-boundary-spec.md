# Aionis Open-Core Boundary Spec

## Goal

Define a written product boundary for the next Aionis release phase:

1. SDK open
2. runtime contracts explicit
3. deeper runtime learning/governance implementation selectively distributed

## Requirements

1. Root product docs should reflect the new boundary.
2. The SDK README should state that it is the primary open developer surface.
3. A dedicated boundary document should explain what is open, what stays controlled, and why.

## Non-Goals

1. Final legal licensing language.
2. Pro plan packaging.
3. Repository restructuring.
