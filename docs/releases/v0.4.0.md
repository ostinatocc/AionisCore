# Aionis Runtime v0.4.0 Lite Developer Preview

Release date target: 2026-04-27

This release moves Aionis Runtime from the v0.3.0 public technical beta line into a **Lite Developer Preview** line.

The release target is the local-first Lite runtime and public TypeScript SDK. It is not a production hosted control plane and does not claim multi-tenant production governance.

## Highlights

1. Hardened the four Runtime layers: Contract Compiler, Trust Gate, Orchestrator, and Learning Loop.
2. Added release-readiness evidence for build, Lite tests, package dry-runs, docs build, and live smoke.
3. Expanded the public SDK host execution-memory facades for task start, execution outcome storage, and workflow contract retrieval.
4. Added Runtime authority boundary inventory, legacy-access inventory, and CI guards that prevent route and host layers from bypassing Runtime seams.
5. Strengthened outcome-contract and execution-evidence gates so stable/authoritative reuse requires explicit evidence.
6. Added service lifecycle and after-exit proof paths for tasks where runtime success must survive agent exit.
7. Added dogfood proof scripts and reporting surfaces for real task-family validation.
8. Tightened public route contracts, planner packets, action retrieval, tool selection, policy memory, semantic forgetting, and rehydration surfaces.

## Install

Start the local Lite runtime:

```bash
npx @ostinato/aionis-runtime start
```

Install the public SDK:

```bash
npm install @ostinato/aionis
```

## Release Boundary

This is a developer-preview release for:

1. local runtime startup and health
2. memory write, handoff, recall, planning context, and action retrieval
3. replay, playbook promotion, workflow anchors, and anchor-guided rehydration
4. local automation and sandbox paths
5. SDK host execution-memory integration

This release does not claim:

1. hosted server control plane readiness
2. public-network hardening
3. production multi-tenant auth, quota, audit, or governance
4. long-running production observability guarantees

For the detailed readiness record, see `docs/RELEASE_READINESS.md`.
