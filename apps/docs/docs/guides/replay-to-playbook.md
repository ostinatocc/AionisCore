# Guide: replay to playbook

If you want to understand where Aionis goes beyond memory storage, start here.

## Scenario

You have a successful repair or operating path and want it to become reusable.

## Flow

1. start a replay run
2. record step before / after transitions
3. end the run
4. compile a playbook from the run
5. evaluate, promote, or repair the playbook
6. reuse that playbook through future guidance or local automation

## Why this matters

This flow is the bridge between:

- remembering execution

and:

- operationalizing execution

That transition is one of the central product bets in Aionis.
